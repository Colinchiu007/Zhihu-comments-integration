# 知乎评论整合工具 - PRD

## 1. 项目概述

### 1.1 产品定位
抓取知乎指定内容的评论，进行情感分析和关键词提取，生成结构化文章。

### 1.2 目标用户
需要分析知乎舆论、提取用户观点的内容创作者和研究人员。

## 2. 核心需求

### 2.1 支持的URL类型

| 类型 | URL格式 | 说明 |
|------|---------|------|
| 文章页 | `zhuanlan.zhihu.com/p/{id}` | 知乎专栏文章 |
| 回答页 | `zhihu.com/question/{qid}/answer/{aid}` | 指定回答（**主要场景**）|
| 问题页 | `zhihu.com/question/{qid}` | 问题页（取第一个展开的回答）|

### 2.2 评论抓取范围（关键）

**核心逻辑：只抓取目标回答下的评论，不是所有回答的评论。**

```
问题页
├── 回答1（默认展开）→ 只抓取这里的评论 ✓
├── 回答2（折叠）
├── 回答3（折叠）
└── ...
```

**实现方式：**
1. 解析URL判断类型
2. 定位目标回答元素（`.AnswerItem`）
3. 在目标回答范围内查找评论（`.CommentItem`）
4. 回答页直接定位该回答的评论
5. 文章页直接抓取所有评论

### 2.3 评论数据结构

```json
{
  "author": "用户名",
  "content": "评论内容",
  "likes": "点赞数",
  "timestamp": "时间戳"
}
```

## 3. 安全机制

### 3.1 反爬虫策略

| 策略 | 实现方式 |
|------|----------|
| 代理轮换 | 支持自定义代理列表 + 免费API |
| 随机延迟 | 3-6秒随机间隔 |
| 浏览器指纹 | 隐藏webdriver属性、模拟真实UA |
| 行为模拟 | 人性化滚动、随机暂停 |
| 验证码处理 | 自动检测并等待人工处理 |

### 3.2 限制策略

- 每日请求上限：1000次
- 单次最大评论：500条
- 失败重试：3次
- 重试间隔：30-60秒

## 4. 功能模块

### 4.1 抓取模块（scraper）
- URL解析与类型识别
- 页面加载与反检测
- 评论定位与提取
- 代理管理

### 4.2 分析模块（analyzer）
- 情感分析（snownlp）
- 关键词提取（jieba）
- 词频统计
- 观点聚类

### 4.3 生成模块（generator）
- 模板化文章生成
- 摘要、关键词、正负面观点

## 5. 使用方式

### 5.1 命令行
```bash
# 抓取指定回答的评论
python main.py --url "https://www.zhihu.com/question/123456/answer/789012" --max-comments 50

# 抓取文章评论
python main.py --url "https://zhuanlan.zhihu.com/p/123456789" --output result.md
```

### 5.2 配置页面
```bash
python start_config.py
# 访问 http://localhost:8501
```

## 6. 输出格式

生成Markdown文章，包含：
- 概述（总评论数、情感倾向）
- 核心关键词
- 情感分布（正面/负面比例）
- 主要观点（正面/负面各5条）
- 热门讨论（高频词）
- 总结

## 7. 技术栈

- **抓取**：Python + Playwright
- **分析**：jieba + snownlp
- **配置**：Streamlit（Web界面）
- **反检测**：自定义反检测模块
